# 功能规划：采集评论任务分发与释放机制优化

## 📋 概述

**功能目标**：重构采集评论任务分发和释放机制，解决任务卡死、锁定不释放、客户端提交记录丢失等问题

**预期价值**：
- 提高任务处理可靠性
- 减少任务重复处理
- 完善任务状态追踪
- 优化资源分配

**影响范围**：
- 后端API：`server/routes/client.js`
- 服务层：`server/services/harvestScheduler.js`
- 客户端：`xiaohongshu-audit-clients/harvest-client/`
- 数据模型：`server/models/DiscoveredNote.js`

---

## 🚨 当前问题清单

### 1. 锁定超时时间不一致
| 位置 | 超时时间 | 问题 |
|------|----------|------|
| `/harvest/pending` | 10分钟 | 任务获取锁定 |
| `/heartbeat` | 30分钟 | 心跳延长锁定（仅针对ImageReview） |
| `harvestScheduler` | 每5分钟清理 | 定时清理超时锁定 |

**问题**：心跳延长30分钟，但获取任务只有10分钟，客户端处理超时后无法通过心跳延长

### 2. 心跳机制不完善
- `/heartbeat` 只更新 `ImageReview.processingLock`，不更新 `DiscoveredNote.harvestLock`
- 采集评论客户端无法通过心跳延长锁定时间

### 3. 客户端提交记录丢失
- 客户端调用 `/harvest/complete` 时没有传递 `clientId`（已修复）
- 后端没有设置 `harvestedBy` 字段（已修复）
- 统计查询显示"从未提交"

### 4. 任务完成但锁定未释放
- 已修复：9个已完成的任务被锁定，已手动释放
- 根因：客户端没有调用 `/harvest/complete` API

### 5. 超时清理不完整
- `harvestScheduler.js` 只清理 `harvestLock`
- `client.js` 的 `/pending-tasks` 清理 `processingLock`
- 没有统一的超时处理

## 🎯 功能分解

- [ ] **问题1**: 统一锁定超时时间
  - 10分钟太短，客户端处理可能超时
  - 建议改为15-20分钟

- [ ] **问题2**: 完善心跳机制
  - 心跳应支持 `DiscoveredNote.harvestLock`
  - 采集评论客户端可延长锁定

- [ ] **问题3**: 确保任务完成记录
  - 客户端传递 `clientId` ✅ 已完成
  - 后端设置 `harvestedBy` ✅ 已完成
  - 前端显示采集者信息

- [ ] **问题4**: 统一超时清理机制
  - 在 `harvestScheduler.js` 中统一处理所有锁定超时
  - 避免分散在多个API中

- [ ] **问题5**: 添加任务健康检查
  - 检查锁定过久（如超过30分钟）的任务
  - 自动释放并记录异常

## 📐 技术方案

### 方案A：最小改动方案（推荐）

**改动内容**：
1. 统一锁定超时时间为15分钟
2. 心跳支持延长 `harvestLock`
3. 统一超时清理机制

**优点**：改动小，风险低
**缺点**：没有从根本上解决问题

### 方案B：重构方案

**改动内容**：
1. 引入 Redis 分布式锁
2. 实现状态机模式管理任务状态
3. 添加任务处理审计日志

**优点**：从根本上解决问题
**缺点**：改动大，风险高

**修改位置**: `/heartbeat` API (第1354行)

```javascript
// 新增：支持 DiscoveredNote.harvestLock 的心跳延长
if (taskIds && Array.isArray(taskIds) && taskIds.length > 0) {
  const lockTimeoutMinutes = 15;
  const lockedUntil = new Date(now.getTime() + lockTimeoutMinutes * 60 * 1000);

  // 同时更新 ImageReview 和 DiscoveredNote 的锁定
  const [imageReviewResult, discoveredNoteResult] = await Promise.all([
    ImageReview.updateMany(
      {
        _id: { $in: taskIds },
        'processingLock.clientId': clientId
      },
      {
        $set: {
          'processingLock.heartbeatAt': now,
          'processingLock.lockedUntil': lockedUntil
        }
      }
    ),
    DiscoveredNote.updateMany(
      {
        noteId: { $in: taskIds },
        'harvestLock.clientId': clientId
      },
      {
        $set: {
          'harvestLock.lockedUntil': lockedUntil
        }
      }
    )
  ]);

  updatedCount = imageReviewResult.modifiedCount + discoveredNoteResult.modifiedCount;
}
```

### 第4步：添加任务健康检查

**新增**: 定期检查异常任务

```javascript
async checkStuckTasks() {
  const now = new Date();
  const stuckThreshold = new Date(now.getTime() - 30 * 60 * 1000); // 30分钟

  const stuckTasks = await DiscoveredNote.find({
    'harvestLock.lockedUntil': { $lt: stuckThreshold },
    harvestLock: { $exists: true }
  });

  if (stuckTasks.length > 0) {
    console.log(`⚠️ [健康检查] 发现 ${stuckTasks.length} 个卡住的任务`);
    // 自动释放
  }
}
```
