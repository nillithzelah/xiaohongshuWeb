## 二、评论采集指令模板

### 基础版：访问笔记并提取评论

```
用浏览器访问小红书笔记 https://www.xiaohongshu.com/explore/笔记ID，
展开所有评论，提取以下信息：
1. 评论内容
2. 评论者昵称
3. 评论时间
4. 点赞数

以表格形式整理输出。
```

### 进阶版：筛选最近评论

```
用浏览器访问小红书笔记 https://www.xiaohongshu.com/explore/笔记ID，
滚动加载所有评论，只提取发布时间在"1小时内"的评论，
包括评论内容、评论者、时间，整理成JSON格式。
```

### 高级版：AI 分析 + 服务器上报

```
执行以下任务：
1. 用浏览器访问 https://www.xiaohongshu.com/explore/笔记ID
2. 滚动页面加载所有评论
3. 提取1小时内发布的评论
4. 用 AI 分析每条评论，判断是否为潜在客户线索（关键词：被骗、维权、投诉、退款等）
5. 将分析结果以 JSON 格式输出

JSON 格式示例：
{
  "noteUrl": "笔记链接",
  "comments": [
    {
      "author": "评论者昵称",
      "content": "评论内容",
      "time": "发布时间",
      "isLead": true/false,
      "reason": "判断理由"
    }
  ]
}
```

## 三、完整工作流示例

### 任务：采集维权笔记评论线索

```
你是一个评论采集助手，执行以下步骤：

第一步：访问笔记
用浏览器访问 https://www.xiaohongshu.com/explore/目标笔记ID

第二步：加载评论
滚动页面到底部，等待评论加载完成，继续滚动直到所有评论都显示

第三步：提取评论
提取所有评论的：作者、内容、时间、点赞数

第四步：筛选评论
只保留发布时间显示为"刚刚"、"X分钟前"、"1小时前"的评论

第五步：AI 分析
分析每条评论，判断评论者是否可能是：
- 维权受害者（提到被骗、投诉、退款等）
- 有消费纠纷的人（提到商家、服务、质量等）

第六步：输出结果
以结构化格式输出潜在线索
```

## 四、注意事项

### 1. 如果遇到登录页面
```
小红书可能要求登录才能查看评论。
解决方案：
- 在浏览器中手动登录小红书
- 导出 Cookie 给 OpenClaw 使用
- 或者使用 Cookie 池
```

### 2. 如果被反爬检测
```
如果访问被拒绝或返回异常，尝试：
1. 添加随机延迟
2. 模拟人类滚动行为
3. 更换访问时间
```

### 3. 限制采集数量
```
每次采集不要超过50条评论，避免触发风控
```

## 六、与现有系统集成

### 发送结果到服务器

```
发送 HTTP POST 请求到 https://www.wubug.cc/xiaohongshu/api/openclaw/harvest，
请求体为：
{
  "noteUrl": "笔记链接",
  "comments": [...评论数组...],
  "clientId": "openclaw-client-1"
}
```
